# みのり日和ファーム デモ管理者マニュアル

## 管理画面を開く

利用者画面右上の「管理画面を試す」を押します。デモではログインせずに操作できます。実運用版ではFirebase Authenticationによる管理者認証を追加する予定です。

## 最初に見る場所

「本日の運営」には次の4項目があります。

| 項目 | 確認すること |
|---|---|
| 予約人数 | 本日または直近の開催枠へ来園する人数 |
| 残席 | 追加予約を受けられる人数 |
| 待機 | 手動繰り上げを検討するグループ数 |
| 通知失敗 | 内容確認と再送が必要な件数 |

## 電話予約を登録する

1. 「電話予約を登録」を押す。
2. 開催枠と大人・子ども・幼児の人数を選ぶ。
3. メールを受け取る想定なら「メール通知あり」を選ぶ。
4. 「電話予約を登録」を押す。

デモでは代表者情報を固定ダミー値で保存します。

## 開催枠を調整する

「開催枠」から次を操作できます。

- 開催枠の新規作成、編集、複製、削除
- 定員の `− / ＋`
- 受付停止・再開
- 開催中止

### 開催枠を作成する

1. 「開催枠」画面の「開催枠を作成」を押す。
2. 体験プラン、単日または毎週、開催日、開始・終了時刻を選ぶ。
3. 定員、年齢別料金、受付期間、キャンセル期限、残りわずか基準、備考を確認する。
4. 内容と作成件数を確認し、「下書き保存」または「公開する」を選んで保存する。

毎週作成は開始日と同じ曜日で終了日まで生成し、一度に最大12枠です。料金や定員は、同じ体験プランの直近設定を初期値にします。

### 公開・編集・削除のルール

- 下書きは管理画面だけに表示され、予約候補には出ません。
- 公開すると利用者カレンダーへ即時反映されます。
- 「複製」は設定だけを引き継ぎ、予約、待機、開催中止状態は引き継ぎません。
- 予約・待機履歴がない枠は、全項目の編集、下書き化、削除ができます。
- 履歴がある枠は、体験、日時、公開状態、削除を変更できません。定員、料金、受付条件、備考は変更できます。
- 履歴がある枠を取り消す場合は「開催中止」を使います。

開催中止では理由を入力します。対象の確定予約・待機申請を中止状態へ変更し、通知プレビューと操作履歴を作成します。

## キャンセル待ちを繰り上げる

1. 「待機者」を開く。
2. 現在の残席とグループ人数を確認する。
3. 残席内なら「予約へ繰り上げる」を押す。

自動繰り上げは行いません。大きなグループを飛ばすかどうかを農園が判断できる設計です。

## 通知を再送する

「通知」を開き、送信失敗サンプルの「再送する」を押します。販売デモでは外部メールを送らず、状態だけを「送信済み（デモ）」へ変更します。

## 操作履歴を確認する

「操作履歴」で、開催枠の作成・編集・削除、予約、キャンセル、電話予約、受付変更、待機繰り上げ、中止、通知再送を時系列で確認できます。

## 初期データへ戻す

画面上部の「初期データへ戻す」を押し、確認画面で確定します。予約・待機・開催枠・通知・履歴が現在月を基準にした初期サンプルへ戻ります。

## デモで入力してはいけない情報

販売デモには、顧客の本名、実在する電話番号、実在するメールアドレスを入力しないでください。動作確認には次を使用します。

```text
氏名: デモ利用者
電話番号: 000-0000-0000
メール: demo@example.invalid
```
