## 確認証拠

### 確認済み

- `npm run lint`: TypeScript型検査成功。
- `npm test`: 27件成功。公開・下書き境界、原子的な一括作成、週次上限、履歴付き編集削除、予約・待機・通知の状態遷移、開催中止の終端性、一覧CTAの整合、決定表、De Morgan則、正規表現の境界を含む。
- `npm run build`: Vite本番ビルド成功。
- 外部Codexレビュー: 初回指摘の開催中止枠への待機登録と、再レビュー指摘の開催中止状態の迂回更新を修正。最終レビューは`ok: true`、blocking 0件。
- 保存形式: バージョン4。デモ初期化で追加枠・予約・待機を初期状態へ戻す。
- 1280px: 管理一覧と3ステップ入力に横ずれなし。モーダル幅820px、ページ横スクロールなし。
- 390px: 管理一覧、開催日時、確認・公開画面に横ずれ・文字重なりなし。カード幅とスクロール幅が一致。
- 公開総合フロー: 管理画面で公開した枠が7月カレンダーへ即時反映され、2名予約後に管理画面が「予約2名・残り10席」へ更新。
- 下書き総合フロー: 管理画面には表示されるが利用者カレンダー・体験一覧には表示されないことを確認。
- 編集・削除: 予約後は体験・日時入力と削除が無効、履歴のない下書きは削除可能。
- アクセシビリティ: モーダル初期フォーカス、Escape終了、呼び出し元へのフォーカス復帰を実ブラウザで確認。
- ブラウザ品質: 画像読み込み失敗0件、console error/warning 0件。
- 開催中止の終端性: 中止済み枠の「停止」「中止」は画面上で無効。通常更新からの中止、中止後の状態復帰、二重中止はRepositoryでも拒否。
- 利用者CTA: 生育調整中・受付停止・開催中止・受付開始前・受付終了は、残席表示とCTAを状態名へ切り替え、一覧から進めないことをPC・390pxで確認。

### 未確認

- 実ブラウザでの毎週3件一括保存。週次生成・終了日包含・12件上限は自動テストで確認済み。

### 対象外

- 本番DBの同時実行制御、バックアップ、データ移行、負荷試験。
- 本物の認証、決済、個人情報、外部メール、監視。
- 遠隔地通信、帯域制限、タイムアウト再試行。

## ロールバック・復旧

- デモデータの復旧: 管理画面の「初期データへ戻す」で現在月基準の初期状態を再生成する。
- 保存形式の復旧: バージョン不一致または破損JSONを検出すると初期状態へ戻す。
- コードのロールバック: 公開前にGit差分を確定し、イベント作成機能のコミット単位で戻せる状態にする。
- 本番データ移行は対象外のため、実データのバックアップ／復元手順はフェーズBで定義する。

## 次の品質ゲート

- 実行項目: 販売デモとしての利用者受入確認。実運用化へ進む場合はフェーズBの設計審査を別途行う。
